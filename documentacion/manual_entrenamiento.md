# Manual de Entrenamiento – Sistema Lean de Auditoría Inteligente

Este manual explica cómo operar el sistema de revisión documental basado en Lean.

## Principios aplicados
- Flujo continuo: No se procesan lotes; cada archivo sigue su camino en cuanto llega.
- Jidoka: El sistema detiene el flujo cuando detecta un defecto.
- Poka-Yoke: Reglas digitales para evitar errores humanos.
- Estandarización: Todos los documentos siguen la misma estructura.
- Mejora continua: Cada fallo se convierte en una oportunidad de análisis.

## Objetivos del operario
- Revisar excepciones señaladas por el sistema
- Confirmar o corregir decisiones automáticas
- Mantener integridad documental
- Registrar errores recurrentes para análisis de causa raíz

## Flujo de trabajo del operario
1. Abrir la bandeja de “Documentos en excepción”
2. Revisar:  
   - Documentos sin texto  
   - Documentos corruptos  
   - Documentos con ID incoherente  
3. Resolver y comentar
4. Enviar retroalimentación
5. Documentar si es un caso crítico

## Reglas claras
- No corregir manualmente archivos si no se documenta el motivo
- Nunca modificar ID sin evidencia
- Todo archivo crítico requiere dos revisiones
- Las mejoras se discuten semanalmente

Este manual permite entrenar a nuevos usuarios y sostener la calidad del sistema.
